Цель игры Виртуальная машина IDE yBasic Операторы

Цель игры

Игрок должен помочь герою разобраться в происходящем. Для того, чтобы прояснить события, требуется связать изменения, произошедшие в ОЗУ виртуальной машины с изменениями, произошедшими на виртуальном экране. Как это сделать? - восстановить программу, производящую эти изменения. Программы в игре написаны на простом языке программирования Yocto BASIC и могут быть частично известны игроку в начале уровня. Все изменения, производимые неизвестной герою программой (сны героя), видны игроку в виде "снапшотов" - эпизодов состояний ОЗУ и экрана виртуальной машины в различные моменты времени. Это похоже на отладку, только наоборот. Переверните свой мир - побудьте немного компьютером! Самое время ответить себе на вопрос - когда я пишу программу, я действительно понимаю, КАК это всё работает?

Особенности архитектуры виртуальной машины

Основой архитектуры виртуальной машины является интерпретатор, который построчно выполняет операторы языка программирования yBASIC. В момент исполнения строки программы интерпретатор может осуществить одно из следующих действий:

  • создать (изменить) переменную или массив
  • изменить порядок выполнения строк программы с записью или без записи в стек
  • произвести печать содержимого переменной или результат выражения на экран
  • произвести ввод значения из виртуальной консоли
  • остановить выполнение программы

ОЗУ виртуальной машины представляет собой массив, размером 256 байт и имеет адреса [0..255]. Условно ОЗУ можно разделить на область переменных и область стека. Область переменных начинается с адреса 0 и её размер растёт в сторону увеличения адресов. Область стека начинается с адреса 255 и её размер растет в сторону уменьшения адресов. В случае пересечения областей переменных и стека возникнет ошибка выполнения программы. При создании переменных, вводе из виртуальной консоли, а также вызове подпрограмм содержимое ОЗУ изменяется.

У виртуальной машины имеется возможность получить данные извне - виртуальная консоль. До начала выполнения интерпретатором программы консоль уже должна содержать абсолютно все необходимые программе данные. Данные из консоли выбираются последовательно при выполнении интерпретатором очередной команды ввода. Если при выполнении команды ввода данных в консоли нет, то возникнет ошибка выполнения программы.

Виртуальная машина имеет экран, работающий в текстовом режиме 80x25. Интерпретатор, выполяя программу, может изменять содержимое виртуального экрана. Вводимая с виртуальной консоли информация также видна на экране, но предваряется символом < . Обычная печать происходит без дополнительных признаков.

IDE (среда разработки и отладки)

Чтобы помочь герою разобраться в происходящем, игроку надо выполнить следующие действия:

  • просмотреть эпизоды сна
  • попытаться восстановить программу, которая производит эти действия
  • отладить полученный код
  • проверить, что написанная программа действительно снится герою

Всё это позволяет выполнить IDE.

Список элементов управления:

Код ОЗУ Консоль Экран Сон Кнопки Меню
Редактор кода и окно сообщений.

Редактор кода позволяет написать свой программный код. В начале уровня может содержать часть програмы, которую требуется восстановить. Окно сообщений отображает синтаксические ошибки и ошибки выполнения программы. При пошаговом выполнении программы в окне сообщений видно, какая строка исполнялась последней. Кнопка CODE показывает, что в данный момент виден именно редактор кода и позволяет переключиться на отображение содержимого ОЗУ.

Окно отображения содержимого ОЗУ.

Окно состояния ОЗУ виртуальной машины позволяет просмотреть области переменных и стека При пошаговом выполнении программы изменения, произведенные последним выполненным оператором, будут окрашены в красный цвет. Кнопка RAM показывает, что в данный момент видно именно окно состояния ОЗУ и позволяет переключиться на редактор кода.

Окно отображения содержимого виртуальной консоли.

Окно виртуальной консоли позволяет задать данные, которые после выполнении оператора вввода попадут в ОЗУ. Можно вводить числовые и строковые данные. Строковые данные должны быть записаны в двойных кавычках. Кнопка INPUT показывает, что в данный момент видно именно окно виртуальной консоли и позволяет переключиться на просмотр экрана виртуальной машины.

Окно отображения содержимого экрана виртуальной машины.

Окно экрана виртуальной машины позволяет увидеть информацию, которую программа выводит с помощью оператора вывода. Также на экране отображаются данные, введенные из консоли. При пошаговом выполнении программы изменения на виртуальном экране происходят только после выполнения операторов вывода или ввода. Кнопка SCREEN показывает, что в данный момент видно именно окно экрана и позволяет переключиться на окно виртуальной консоли.

Элементы просмотра и управления эпизодами сна.

Элементы просмотра сна позволяют увидеть отрывки из сна героя, по которым игрок должен восстановить неизвестную герою программу. На панели сна есть состояние ОЗУ, состояние экрана и кнопки управления сном. Кнопки позволяют перемещаться по эпизодам сна:

  • [< - Перейти в начало сна
  • << - Перейти к предыдущему изменению состояния ОЗУ во сне
  • < - Перейти к предыдущему отрывку сна
  • > - Перейти к следующему отрывку сна
  • >> - Перейти к следующему изменению состояния ОЗУ во сне
  • >] - Перейти в конец сна
  • RESET - Просмотреть сон повторно
Состояния ОЗУ и экрана при нажатии какой-либо кнопки изменяются синхронно. Изменения, произошедшие в ОЗУ после нажатия кнопки, отображены красным цветом. Если программа во сне использовала неизвестные игроку входные данные, то после нажатия кнопки RESET они будут изменены. В поле SNAPSHOT показано, какой конкретно отрывок сна виден в данный момент и сколько всего эпизодов в данном сне. Кнопка CLOSE закрывает панель сна.

Кнопки управления интерпретатором и ходом игры.

Кнопки позволяют запускать и отлаживать программу на yBasic, а также управлять игрой:

  • RUN - Запустить программу на выполнение
  • STEP - Запустить выполнение на один шаг
  • RESET - При пошаговом выполнении вернуться к началу программы
  • SHOW - Показать панель управления сном
  • CHECK - Проверить, верно ли игрок угадал программу из сна
  • NEXT - Перейти к следующему уровню (может быть не видна)
После нажатия кнопки CHECK происходит сравнение программы, написанной пользователем, с программой, которую необходимо выяснить герою на данном уровне. Сравнение не учитывает особенности написания кода и нумерации строк (кроме случаев, когда номер строки помещается в стек). Важным является только точное воспроизведение последовательности и сути действий, которые совершает программа из сна в ОЗУ и на экране. Эти действия и их очередность видны на панели сна. Если пользователь верно воспроизвёл программу, то после проверки в окне сообщений возникнет соответствующая надпись и кнопка NEXT станет доступна и позволит перейти на следующий уровень. Если программы не совпадут, в окне сообщений можно будет прочесть, в чем именно они не совпали.

Меню настроек.

Меню настроек позволяет изменять элементы интерфейса и игрового процесса:

  • В ОЗУ отображать строковые переменные как набор символов, а не коды ASCII
  • Числовые данные представлять в ОЗУ с помощью шестнадцатиричных чисел
  • Показывать/скрыть часть программы, которую требуется восстановить
  • Показывать/скрыть подсказки по действиям элементов управления
  • Проигрывать/остановить фоновую музыку
  • Показывать панель сна на одном экране с панелью кода (только для широких экранов)
Если применить изменения настроек, то IDE перезагрузится и прогресс уровня будет утерян. С помощью настроек можно значительно усложнить игровой процесс (по умолчанию настроен самый простой вариант).

Язык программирования Yocto BASIC (yBasic)

Программа на yBasic представляет собой набор строк, каждая из которых начинается с номера. Номер каждой последующей строки должен быть больше предыдушего. Программу выполняет интепретатор. После выполнения интерпретатором последней строки из списка программа завершается. На каждой строке может быть только один оператор языка (исключение составляет только оператор условного выполнения IF...[ELSE]). Здесь и далее при описании операторов в квадратных скобках указываются необязательные параметры, а в треугольных скобках указываются обязательные параметры.

Пример корректно написанной программы: 

0 REM FIRST LINE 
10 REM SECOND LINE
100 REM THIRD LINE

В такой программе интерпретатор считывает последовательно строки в сторону увеличения порядковых номеров. Работа программы завершится после обработки строки с номером 100.

Операторы yBasic

Список операторов yBasic по категориям:

Комментарии Досрочное завершения выполнения Создание и изменение переменных Операторы ввода-вывода Операторы нелинейного выполнения

Список операторов yBasic по названию:

REM END LET = DIM INPUT PRINT[LN] GOTO IF...[ELSE] GOSUB RETURN

REM [КОММЕНТАРИЙ]: оператор комментария. Cтроки, которые начинаются с REM, считываются, но не исполняются интерпретатором

END: оператор досрочного завершения программы. При выполнении END программа завершается даже если этот оператор расположен не на последней строке программы

Пример использования оператора END: 

0 REM FIRST LINE
10 REM SECOND LINE
50 END
100 REM THIRD LINE

В такой программе строка 100 обработана уже не будет. Выполнив оператор END на строке 50, интерпретатор прекратит работу.

LET <ИМЯ ПЕРЕМЕННОЙ> [ = НАЧАЛЬНОЕ ЗНАЧЕНИЕ]: оператор создания переменной. Можно создавать числовые и строковые переменные. Все переменные имеют имена из одной буквы латинского алфавита, то есть возможно создать до 26 переменных. Таким образом, корректные имена переменных: A, B, C ...Z. В качестве начального значения могут быть использованы как одиночные значения, так и выражения.

Выражения для числовых переменных должны содержать только числа, математические и битовые операторы (+, -, *, /, NOT, XOR, OR, AND). Числа могут быть представлены в десятичном и шестнадцатиричном форматах: -1, 2, 3 - десятичные числа. &HFF, &H01, &H1A - шестнадцатиричные числа. (NOT &HE3 + 100) / 2 - это числовое выражение, результатом выполнения которого будет число -64.

Выражения для строковых переменных должны содержать только строковые значения (записываются в кавычках) и оператор конкатенации(+). "HELLO " + "WORLD!" - это строковое выражение, результатом которого будет строка "HELLO WORLD!".

Числовые переменные могут хранить 1 байт информации и представляют собой целые знаковые числа. В памяти числовая переменная также занимает 1 байт. Числовая переменная может принимать значения [-128...127]. Переменные с именами A, B, C в тексте программы будут интерпретированы как числовые.

Строковые переменные могут хранить строку длинной до 8 символов. В памяти такая переменная занимает 9 байт - по одному байту на каждый возможный символ строки и один дополнительный нулевой байт - признак конца строки. Однобуквенное имя строковой переменной должно оканчиваться на символ $. Переменные с именами A$, B$, C$ в тексте программы будут интерпретированы как строковые.

Пример использования оператора LET: 

10 REM CREATING A NUMERIC VARIABLE A WITH INITIAL VALUE 0
11 LET A 
20 REM CREATING A NUMERIC VARIABLE В WITH INITIAL VALUE 127
21 LET B = &H7F
30 REM CREATING A NUMERIC VARIABLE C WITH INITIAL VALUE А + В (0 + 127)
31 LET C = A + B
40 REM CREATING A STRING VARIABLE D$ WITH INITIAL VALUE "0123456" 
41 LET D$ = "0123456"
50 REM CREATING AN UNINITIALIZED VARIABLE E$ (EMPTY STRING)
51 LET E$
60 REM CREATING A STRING VARIABLE F$ WITH INITIAL VALUE D$ + E$ ("0123456" + "")
61 LET F$ = D$ + E$

Значения переменных можно изменить с помощью оператора присваивания =.

Пример использования оператора присваивания: 

0 LET A
1 LET C$
2 LET D$
3 LET E$
10 REM ASSIGMENT THE VALUE 66 TO A NUMERIC VARIABLE A
11 A = 33 + 33
20 REM ASSIGMENT THE VALUE "123456" TO A STRING VARIABLE C$
21 C$ = "123456"
30 REM ASSIGMENT THE VALUE "7890" TO A STRING VARIABLE D$
31 D$ = "7890"
40 REM ASSIGMENT THE VALUE "1234567890" (C$ + D$) TO A STRING VARIABLE E$
41 E$ = C$ + D$
42 REM BUT NOW VARIABLE E$ IS EQUAL "12345678" BECOUSE STRING VARIABLE LENGTH
43 REM IS ONLY 8 SIMBOLS

DIM <ИМЯ МАССИВА>(<РАЗМЕРНОСТЬ МАССИВА: 1..8>): оператор создания массива. Можно создавать числовые и строковые массивы. Каждый элемент массива представляет собой числовую или строковую переменную, соответственно. Максимальное количество элементов массива 8, минимальное - 1. В памяти максимальный числовой массив займёт 1 * 8 = 8 байт, а максимальный строковый массив займёт 9 * 8 = 72 байта. Элементы массива всегда нумеруются с нуля. Для обращения к элементу массива требуется в круглых скобках указать индекс элемента, к которому происходит обращение. Для массивов работают такие же правила именования, как для соответствующих переменных. Для элементов массивов работают такие же правила присваивания, как для соответствующих переменных.

Пример использования DIM: 

0 LET D = 3
1 LET E = 1
3 LET F$ = "1234"
10 REM CREATING A NUMERIC ARRAY A WITH 3 ELEMENTS (3 BYTES IN RAM)
11 DIM A(3)
20 REM CREATING A STRING ARRAY B$ WITH 5 ELEMENTS (45 BYTES IN RAM)
21 DIM B$(5)
30 REM CREATING A NUMERIC ARRAY C WITH 5 (D + E + 1) ELEMENTS (5 BYTES IN RAM)
31 DIM C(D + E + 1)
40 REM ASSIGMENT THE VALUE "12345678" (F$ + "5678") TO ELEMENT 4 (D + E) OF A STRING ARRAY B$
41 B$(D + E) = F$ + "5678"
50 REM ASSIGMENT THE VALUE 3 (D) TO ELEMENT 1 (E) OF A NUMERIC ARRAY A
51 A(E) = D
60 REM ASSIGMENT TO A NUMERIC VARIABLE E ELEMENT 1 (E) OF A NUMERIC ARRAY A
61 E = A(E)
62 REM NOW VALUE OF A NUMERIC VARIABLE E IS 3
70 REM ASSIGMENT TO A STRING VARIABLE F$ ELEMENT 4 OF A STRING ARRAY B$
71 F$ = B$(4)
72 REM NOW VALUE OF A STRING VARIABLE F$ IS "12345678"

INPUT <ИМЯ ПЕРЕМЕННОЙ ИЛИ МАССИВА>: оператор ввода данных из виртуальной консоли в переменную или элемент массива. Можно вводить числовые и строковые данные. Для успешного ввода необходимо, чтобы виртуальная консоль содержала вводимые данные.

Пример использования INPUT: 

0 LET A
1 LET B$
3 DIM C(3)
4 DIM D$(5)
10 REM INPUT TO A NUMERIC VARIABLE A
11 INPUT A
12 REM NOW THE VALUE OF VARIABLE A IS EQUAL TO CONSOLE INPUT DATA
20 REM INPUT TO A STRING VARIABLE B 
21 INPUT B$
22 REM NOW THE VALUE OF VARIABLE B$ IS EQUAL TO CONSOLE INPUT DATA
30 REM INPUT TO AN ELEMENT OF A NUMERIC ARRAY C 
31 INPUT C(1)
32 REM NOW THE VALUE OF ELEMENT 1 OF ARRAY C IS INPUTED FROM THE CONSOLE
40 REM INPUT TO AN ELEMENT OF A STRING ARRAY D$
41 INPUT D$(0)
42 REM NOW THE VALUE OF ELEMENT 0 OF ARRAY D$ IS INPUTED FROM THE CONSOLE

Пример данных, поступающих из консоли: 

123
"ABCD"
-123
"DCBA"

PRINT[LN] [ИМЯ ПЕРЕМЕННОЙ, ЭЛЕМЕНТА МАССИВА ИЛИ ВЫРАЖЕНИЕ]: оператор вывода данных на виртуальный экран. Можно выводить числовые и строковые переменные, элементы массива и результаты вычисления выражений. Если оператор используется с суффиксом LN, то после вывода данных будет осуществлён переход на новую строку. Числа выводятся в десятичном формате.

Пример использования PRINT: 

0 LET A = &H0A
1 LET B$ = " ABCD "
3 DIM C(3)
4 C(1) = -10
5 DIM D$(5)
6 D$(4) = " DCBA "
10 REM PRINTING THE VALUE OF A NUMERIC VARIABLE A (10)
11 PRINT A
20 REM PRINTING THE VALUE OF A STRING VARIABLE B (" ABCD ") 
21 PRINT B$
30 REM PRINTING THE VALUE OF AN ELEMENT 1 OF ARRAY C (-10)
31 PRINT C(1)
40 REM PRINTING THE VALUE OF AN ELEMENT 4 OF ARRAY D$ (" DCBA ")
41 PRINT D$(4)
50 REM PRINTING THE LINE FEED
51 PRINTLN
60 REM PRINTING THE STRING EXPRESSION
61 PRINTLN "THIS IS B$ + D$(4) RESULT:" + B$ + D$(4)
70 REM PRINTING THE NUMERIC EXPRESSION
71 PRINT "THIS IS A + C(1) + 1 RESULT: "
72 PRINTLN A + C(1) + 1

Результат вывода на экран: 

10 ABCD -10 DCBA 
THIS IS B$ + D$(4) RESULT: ABCD  DCBA 
THIS IS A + C(1) + 1 RESULT: 1

GOTO <НОМЕР СТРОКИ>: оператор передачи управления на строку с указанныи номером.

Пример использования GOTO: 

10 LET A = 1
20 LET B = 2
30 REM PROGRAM EXECUTION AFTER LINE 31 MOVES TO LINE 50 INSTED OF LINE 40
31 GOTO 50
40 PRINT A
50 PRINTLN B
51 REM NOW ONLY THE VALUE 2 IS DISPLAYED ON THE SCREEN

IF <УСЛОВИЕ> THEN <ОПЕРАТОР 1> [ELSE <ОПЕРАТОР 2>]: оператор ветвления. Позволяет выполнить один оператор в случае выполнения заданного условия и опционально выполнить другой оператор, если заданное условие не выполняется. Условие может компоноваться из нескольких выражений c помощью операторов сравнения < (левая часть меньше), > (левая часть больше), = (равно), <> (не равно) и логических операторов AND (логическое "И"), OR (логическое "ИЛИ"), NOT (логическое "НЕ"), XOR ("исключающее ИЛИ")

Пример логических выражений: 

(A + B) * 2 < C + D
NOT ((A + B) * 2 <> C + D) XOR (A > C)

Пример использования IF...[ELSE] (организация цикла): 

10 LET I = 10
20 I = I - 1
30 PRINT I
40 REM GOTO WORKS IF THE VALUE OF THE NUMERIC VARIABLE I IS NOT EQUAL TO 0
41 IF NOT (I = 0) THEN GOTO 20 ELSE PRINTLN " THE END!"

Результат выполнения цикла (содержимое экрана): 

9876543210 THE END!

GOSUB <НОМЕР СТРОКИ>: оператор вызова подпрограммы. При выполнении этого оператора номер текущей строки (адрес возврата) помещается в память (в область стека). Выполнение программы продолжится со строки, номер которой указан в качестве параметра к GOSUB. Адрес возврата в стеке занимает 2 байта.

RETURN: оператор возврата из подпрограммы. При выполнении этого оператора из стека извлекается адрес возврата и выполнение программы продолжится со строки, номер которой был извлечен из стека. Область памяти, содержащая извлеченный номер строки очищается (происходит обнуление двух байт, занимаемых адресом возврата).

Пример использования GOSUB и RETURN: 

0 PRINTLN "THIS IS PRINTED BEFORE EXECUTING A SUBROUTINE"
10 REM AFTER EXECUTION OF LINE 11 NEXT LINE NUMBER IS 40
11 GOSUB 40
20 PRINTLN "THIS IS PRINTED AFTER EXECUTING A SUBROUTINE"
30 END
40 PRINTLN "THIS IS PRINTED FROM A SUBROUTINE"
50 RETURN
51 REM NOW EXECUTION WILL RETURN TO THE LINE 20

Результат вывода на экран: 

THIS IS PRINTED BEFORE EXECUTING A SUBROUTINE
THIS IS PRINTED FROM A SUBROUTINE
THIS IS PRINTED AFTER EXECUTING A SUBROUTINE
